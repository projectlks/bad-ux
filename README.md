# 🏛️ The Museum of Bad UX

A curated collection of the worst, most frustrating, and infuriating User Interfaces (UI) ever created. 

The main goal of this project is to serve as a reminder of **"what never to do"** in UI design, and to serve as a fun way to troll fellow developers. The museum currently features a wide variety of cursed, over-engineered volume sliders.

## 🎢 Current Exhibits (The Hellscapes)

1. **Gravity Volume (`/volume/gravity`)**: A slider that constantly falls back to zero due to gravity. Catch it if you can.
2. **Momentum Slider (`/volume/momentum`)**: A slider with slippery ice physics. Targeting an exact number is completely impossible.
3. **100 Sliders Average (`/volume/hundred`)**: An exhausting system where you must drag all 100 individual sliders to their maximum to achieve 100% volume.
4. **Drag & Drop Pile (`/volume/dragdrop`)**: Drag exactly 50 sound wave icons into a box to get 50% volume.
5. **Binary Volume (`/volume/binary`)**: Toggle 7 bits (0s and 1s) to set your volume. 1100100 = 100%. For programmers only.
6. **Calculator Volume (`/volume/calculator`)**: Manually calculate your desired volume. Don't divide by zero.
7. **Dice Roll Volume (`/volume/dice`)**: Roll the dice to determine your volume. Feeling lucky today?
8. **Loading Bar Volume (`/volume/loading`)**: Wait for the volume to load. Stop it exactly at your desired percentage.
9. **Vertical Slider (`/volume/vertical`)**: A perfectly normal vertical slider... or is it hiding a dark secret?

## 💻 Local Development

Want to experience this hell on your local machine? Follow these steps:

**1. Clone the repository:**
```bash
git clone [https://github.com/projectlks/bad-ux.git](https://github.com/projectlks/bad-ux.git)
cd bad-ux
```

**2. Install packages:**
```bash
npm install
```

**3. Start the development server:**
```bash
npm run dev
```
Then, open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

# 🏛️ The Museum of Bad UX (မြန်မာဘာသာ)

ကမ္ဘာပေါ်မှာ အဆိုးရွားဆုံး၊ အသုံးချရ အခက်ခဲဆုံးနဲ့ ဒေါသထွက်စရာ အကောင်းဆုံး User Interfaces (UI) များကို စုစည်းပြသထားသော ပြတိုက်ဖြစ်ပါသည်။ 

ရည်ရွယ်ချက်ကတော့ **"ဘယ်တော့မှ ဒီလို UI မျိုး မရေးမိစေဖို့"** နဲ့ Developer အချင်းချင်း နောက်ပြောင် (Troll) ရန်အတွက်သာ ဖြစ်ပါတယ်။ လက်ရှိ ပြတိုက်တွင် အသံအတိုးအလျှော့ (Volume Sliders) အမျိုးအစားပေါင်းများစွာကို ခင်းကျင်းပြသထားပါသည်။

## 🎢 လက်ရှိ ပြသထားသော ငရဲခန်းများ (Exhibits)

1. **Gravity Volume (`/volume/gravity`)**: ကမ္ဘာ့ဆွဲအားကြောင့် အောက်ကို အမြဲပြုတ်ကျနေတဲ့ Slider။ အသံလိုချင်ရင် မပြုတ်ကျအောင် ဖမ်းဆွဲထားရပါမယ်။
2. **Momentum Slider (`/volume/momentum`)**: ရေခဲပြင်ပေါ်က ဆပ်ပြာခဲလို လျှောတိုက်နေမယ့် Slider။ မိမိလိုချင်တဲ့ ဂဏန်းအတိအကျရဖို့ လုံးဝ (လုံးဝ) မဖြစ်နိုင်ပါ။
3. **100 Sliders Average (`/volume/hundred`)**: 100% ရဖို့အတွက် Slider ၁၀၀ လုံးကို အဆုံးထိ လိုက်ဆွဲတင်ရမယ့် အချိန်ကုန်လူပန်းစနစ်။
4. **Drag & Drop Pile (`/volume/dragdrop`)**: အသံ ၁% တက်ဖို့ Icon တစ်ခုကို Box ထဲ ဆွဲထည့်ရပါမယ်။ ၅၀% လိုချင်ရင် အကြိမ် ၅၀ တိတိ ဆွဲထည့်ပါ။
5. **Binary Volume (`/volume/binary`)**: 1 နဲ့ 0 (7 bits) ကိုပဲ သုံးပြီး Volume ကို တွက်ချက်ရပါမယ်။ Programmer များသာ သုံးရန်။
6. **Calculator Volume (`/volume/calculator`)**: မိမိလိုချင်သော အသံပမာဏကို သင်္ချာတွက်ပြီးမှ ရယူပါ။
7. **Dice Roll Volume (`/volume/dice`)**: အသံဘယ်လောက်ရမလဲ ဆိုတာကို အံစာတုံးခေါက်ပြီး ကံစမ်းရပါမယ်။
8. **Loading Bar Volume (`/volume/loading`)**: အသံတက်လာဖို့ကို Loading အပြည့်စောင့်ရပါမယ်။
9. **Vertical Slider (`/volume/vertical`)**: သာမန်ဒေါင်လိုက် Slider လို့ ထင်ရပေမယ့် သုံးကြည့်တဲ့အခါမှ ဒုက္ခပေးမယ့် အရာ။

## 💻 မိမိစက်တွင် Run ကြည့်ရန် (Local Development)

ဒီ ငရဲခန်း Project ကို သင့်စက်ထဲမှာ စမ်းသပ်ချင်တယ်ဆိုရင် အောက်ပါအတိုင်း Run နိုင်ပါတယ်-

**၁။ Project ကို Clone လုပ်ပါ:**
```bash
git clone [https://github.com/projectlks/bad-ux.git](https://github.com/projectlks/bad-ux.git)
cd bad-ux
```

**၂။ Packages များကို Install လုပ်ပါ:**
```bash
npm install
```

**၃။ Development Server စတင်ပါ:**
```bash
npm run dev
```
ထို့နောက် Browser တွင် [http://localhost:3000](http://localhost:3000) သို့ ဝင်ရောက်ကြည့်ရှုပါ။
